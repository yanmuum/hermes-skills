# 工程造价编制技能

## 触发命令

用户在飞书/终端发送 **「工程造价」** 时，启动此技能。

用户需提供：
1. **图纸文件**（PDF扫描件 或 DXF文件）
2. **材料信息价文件**（PDF/Excel）
3. 或直接告知工程参数（长度、断面尺寸等）

## 工作流程

### 第1步：文件获取
- 图纸文件路径：从桌面 /mnt/c/Users/Administrator/Desktop/ 获取
- 复制文件到 ~/ (WSL home)
- 识别文件类型：.pdf（扫描件）或 .dxf（CAD矢量）

### 第2步：PDF提取（如为PDF文件）
1. **尝试 pymupdf 提取文字**
   ```python
   import pymupdf
   doc = pymupdf.open("文件.pdf")
   for page in doc:
       text = page.get_text()
   ```
2. **如果无文字（扫描件）→ OCR**
   - 使用 pymupdf 以 300 DPI 渲染页面
   - 用 OCR.space API 识别（apikey="helloworld", language="chs", OCREngine=2）
   - 如遇 rate limit（429），等待 retryAfter 秒后重试
   ```python
   import requests
   r = requests.post("https://api.ocr.space/parse/image",
       data={"apikey": "helloworld", "language": "chs", "OCREngine": 2},
       files={"file": ("page.jpg", f, "image/jpeg")},
       timeout=60)
   ```

### 第3步：DXF解析（优先使用）

**⚠️ 编码关键：DXF文件必须用 GBK编码 读取（header标注ANSI_936），否则中文乱码**

```python
with open("file.dxf", "r", encoding="gbk", errors="replace") as f:
    lines = f.readlines()
```

**通用解析方法：** 将DXF解析为(code, value)组码对列表，然后扫描实体

```python
pairs = []
i = 0
while i < len(lines):
    line = lines[i].strip()
    if not line: i += 1; continue
    try:
        code = int(line)
        if i + 1 < len(lines):
            pairs.append((code, lines[i+1].strip()))
            i += 2
        else: i += 1
    except ValueError: i += 1
```

**解析各类实体提取关键数据：**

| 实体类型 | 层名 | 提取内容 |
|---------|------|---------|
| TEXT / MTEXT | 任意 | 文字标注、尺寸值 |
| TCH_TEXT | PUB_TEXT / PUB_TAB | 房间名、工程量表内容 |
| TCH_DIMENSION2 | PUB_DIM | 天正尺寸标注（code 42=尺寸值） |
| TCH_WALL | WALL | 墙体（code 42=墙体厚度mm） |
| TCH_OPENING | WINDOW | 门窗（code 40=宽, code 41=高, code 70=类型） |
| TCH_COMPOSING | 任意 | 多行文字说明（用 ^M^J 分隔行） |
| TCH_DRAWINGNAME | DIM_SYMB | 图纸名称标注 |
| TCH_MULTILEADER | DIM_LEAD | 引线标注文字 |
| TCH_RECTSTAIR | STAIR | 楼梯（有=局部多层） |
| 3DFACE | 任意 | 3D面片数据（含实际坐标尺度） |
| LWPOLYLINE | COLUMN/任意 | 柱/轮廓线位置 |
| LINE | PUB_DIM | 尺寸线 |

**关键信息提取目标（根据项目类型不同）：**

**水渠/挡土墙项目：**
- 水沟/渠道总长度（搜索"总长"、总长等关键词）
- 挡土墙类型及分段长度（PUB_TAB层工程量表）
- 断面尺寸（顶宽/底宽/高）
- 材料规格标注 + 设计说明

**厂房/建筑项目：**
- 建筑尺寸（长/宽/高，用户提供或从3DFACE坐标推算）
- 房间布局（PUB_TEXT层房间名：办公室/宿舍/会议室等）
- 墙体厚度（TCH_WALL的code 42值）
- 门窗尺寸（TCH_OPENING的code 40=宽, code 41=高）
- 柱网布置（COLUMN层坐标）
- 楼梯（TCH_RECTSTAIR存在=局部多层）
- 图纸名称（TCH_DRAWINGNAME标识首层/二层等）

### 第4步：信息价数据整理
- 从材料信息价PDF中提取（OCR或文字提取）
- 整理为结构化表格：材料名称、规格、单位、不含税价
- 取最新月份（通常为3月份）价格
- 重点关注：钢筋、水泥、砂石、混凝土、管材等工程常用材料

### 第5步：工程量计算

#### 方案A：水渠/挡土墙项目
根据图纸数据计算各分项工程量：

| 分项 | 计算方式 |
|------|---------|
| 土方开挖 | (墙底宽+0.3×2)×(墙高+0.1+0.5)×总长 |
| 浆砌块石/砌体 | (顶宽+底宽)/2×高×总长 |
| 混凝土垫层 | (底宽+0.1+0.1)×0.1×总长 |
| 钢筋混凝土压顶 | 顶宽×0.15×总长（钢筋约80kg/m³） |
| 土方回填 | 开挖量×0.65（估算） |
| 变形缝 | 总长/15 |
| 泄水孔 | 墙面面积/4（梅花形@2000） |
| 土工布 | 墙高×总长 |
| 砂砾反滤层 | 土工布面积×0.10 |

#### 方案B：钢结构厂房/仓库项目

根据用户提供的建筑尺寸（长×宽×高）和结构参数计算：

**基础工程：**
- 独立基础数量：长方向柱距6m，宽方向柱距6~7m
  - 数量 ≈ (L/6+1) × (W/6+1)
- 每个基础混凝土量 ≈ 1.5×1.5×0.8 = 1.8m³
- 基础钢筋 ≈ 80kg/m³混凝土
- C10垫层 ≈ 1.7×1.7×0.1 × 基础数
- 土方开挖 ≈ 1.8×1.8×1.0 × 基础数 × 1.3(放坡系数)
- 回填土方 = 开挖 - 基础混凝土 - 垫层

**主体钢结构：**
- 总用钢量估算：
  - 轻型钢结构厂房：25~35 kg/m² × 占地面积
  - 含局部二层：再加 15~20 kg/m² × 二层面积
- 分项拆分（按总量比例）：
  - 钢柱：约35%
  - 屋面钢梁：约25%
  - 檩条/C型钢：约22%
  - 二层钢构：约18%（如有）

**围护系统：**
- 彩钢夹芯板墙体面积：(L+W)×2×H - 门窗面积
  - 门窗面积估算：约80~120m²（含卷帘门2500×1800×2+铝合金窗）
- 彩钢板屋面面积：L×W×1.1（含挑檐）

**楼地面工程：**
- 重载混凝土地面：首层仓库面积 × 厚度（200mm）
  - 首层仓库面积 = L×W - 二层投影面积
  - 碎石垫层 = 首层仓库面积 × 0.15（150mm）
- 二层楼板：二层面积 × 0.12（120mm厚）
  - 楼板钢筋 ≈ 80kg/m³

**其他：**
- 钢楼梯+栏杆：如有二层按1项计
- 散水坡道：沿建筑周长×0.6m宽
- 室内轻质隔墙：办公区面积×0.6（估算）
- 门：每间房1樘（900×2100或1500×2100）
- 窗：4000×1800或1800×1500

### 第6步：综合单价分析
- 材料费：套用河源信息价
- 人工+机械：参考市场价估算
- 综合单价 = 材料费 + 人工费 + 机械费 + 管理费

### 第7步：造价汇总

| 费用项 | 计算方式 |
|--------|---------|
| 分部分项工程费 | Σ(工程量×综合单价) |
| 措施项目费 | 约6.5% |
| 其他项目费 | 约3.0% |
| 规费 | 约7.0% |
| 税金 | (前四项之和)×9% |

### 第8步：输出文件

**必须交付的文件（缺一不可）：**

1. **Markdown报告**（.md）— 完整文字版，保存到 ~/ 并复制到桌面
2. **Excel文件**（.xlsx）— 使用 openpyxl 生成，含4~5个工作表：
   - 封面（项目名称、结构类型、面积、编制依据、日期）
   - 工程量清单（序号/项目/单位/工程量/综合单价/合价，用蓝色底色区分章节）
   - 造价汇总（分部分项→措施费→其他费→规费→税金→总造价，总价行用黄色高亮）
   - 材料价格参考（信息价摘录）或 设计参数（项目技术参数）

**文件交付清单（逐一检查）：**
```
✅ cp ~/xxx_工程造价书.md /mnt/c/Users/Administrator/Desktop/
✅ cp ~/xxx_工程造价书.xlsx /mnt/c/Users/Administrator/Desktop/
```
- 必须确认两个文件都存在桌面后再告知用户
- 文件命名格式：`{项目简称}_工程造价书.{md|xlsx}`
- 用户若反馈缺文件，立即补拷并确认

**Excel格式规范：**
- 字体：微软雅黑
- 表头：深蓝底白字（#003366）
- 章节行：浅蓝底（#E8F0FE）
- 合计行：浅黄底（#FFF2CC）
- 总造价：红色加粗（#CC0000）
- 金额格式：'#,##0.00'
- 列宽适配内容
- 所有单元格加细边框

### 第9步：回复用户

- **简洁优先**：直接给出总造价、单方造价、关键数据，不要长篇解释流程
- 如果用户要求排除某些工程，在造价中明确标注"不含XXX"
- 告知桌面已生成的文件列表
- 如果用户问其他问题（如能否做设计），简短回答能力边界后引导回当前任务

## 注意事项

1. DXF文件必须用 **GBK编码** 读取（header标注ANSI_936），不能用UTF-8
2. 天正（TCH_*）实体是加密格式，TCH_DIMENSION2的尺寸值（code 42）需结合上下文解读，TCH_WALL的code 42=墙厚mm
3. TCH_OPENING: code 40=宽mm, code 41=高mm, code 70=类型(0=门,2=窗,3=门联窗), code 71=子类型
4. 工程量表可能有多个版本（同一图纸不同修改阶段），取最新完整版
5. 用户说"其他工程不需要计算"时，只算主体工程，排除项要明确标注
6. **文件交付必须拷贝两份（.md + .xlsx）到桌面，检查确认后再回复**
7. Firebase/OCR.space免费API每天20次限制，超限后等待 retryAfter 秒
8. 回复风格：简洁直接，给总造价+单方造价+关键数据即可，不要解释计算步骤
9. 材料信息价已整理过的文件（河源市2026年第一季度信息价_整理.txt）可直接复用，无需重新OCR
10. 不包含的项目要明确标注排除（如：不含室内装修、给排水、电气、消防）
