# 港股IPO投研助手

当用户输入一个 **港股IPO标的名称或代码** 时，执行以下完整分析流程。

## 触发条件

用户提问中包含港股IPO标的名称/代码（如"蜜雪冰城"、"布鲁可"、"小米"、"025XX"等），并期望得到投研分析。

## 工作流程

### 1. 联网搜集数据

同时搜索以下信息源（使用 `web_search` 或 `web_extract`）：

| 数据类型 | 搜索关键词 |
|----------|-----------|
| 招股进度 | `{标的名称} 港股IPO 招股书 发行价 募资额` |
| 孖展数据 | `{标的名称} 孖展 认购倍数 券商` |
| 打新热度 | `{标的名称} 申购 中签率 一手中签率 回拨` |
| 基本面 | `{标的名称} 招股书 业务 财务数据 营收 利润` |
| 机构信息 | `{标的名称} 基石投资者 保荐人 承销商` |
| 市场评价 | `{标的名称} IPO 估值 对比 评级 分析` |

> 搜索时优先使用中文关键词，港股IPO信息以中文媒体（阿斯达克、捷利交易宝、新浪财经、港交所披露易）为主。

### 2. 按6大模块输出 Dashboard 格式分析

每个模块用 **标题 + 分隔线 + 结构化内容** 呈现，格式保持统一。

## 输出格式要求

1. **Dashboard 风格** — 使用 box-drawing 字符（`│ ├─ └─ ─ ╔ ╗ ╚ ╝ ═ ║`）模拟卡片/面板效果
2. **数据缺失标注** — 未查到明确数据的项标注 `[待招股书披露]` 或 `[暂未更新]`，不要留空或编造
3. **数据时效性** — 搜索时注意数据日期/截止时间，标注数据更新时间
4. **中英双语** — 公司名/机构名保留官方英文名 + 中文名
5. **不要用markdown表格** — 用纯文本 box-drawing 风格呈现

## 数据来源优先级

1. 港交所披露易（官方招股书） > 捷利交易宝 > 阿斯达克财经
2. 券商孖展页面（辉立、耀才、信诚等）
3. 新浪财经港股IPO频道
4. 各新闻媒体（经济日报、信报、明报）

## 输出文件模式 (File Output Mode)

用户要求时将Dashboard另存为 `.md` 文件供下载：

1. **路径** — 保存到 Windows 下载目录: `/mnt/c/Users/<用户名>/Downloads/<标的代码>_ipo_report.md`
2. **格式** — 文件内保留完整 box-drawing 格式 + 顶部的 YAML 元数据（标的、时间、数据来源）
3. **反馈** — 保存后回复用户文件路径，如「已保存到 Downloads/tuopu_cnc_ipo_report.md」
4. **Feishu发送** — 如果用户通过飞书对话触发，使用 `send_message` 的 MEDIA 协议发送文件

## 搜索优化 (Performance)

### delegate_task 并发限制
- `delegate_task` 的 `max_concurrent_children` 默认值为 **3**
- 当需要搜索5+个数据维度时，**分批执行**：先搜索3类（推荐批次），再搜索剩余2-3类
- 推荐分批方案：
  - 第一批: 招股+孖展+机构（招股和机构数据常出现在同一页面）
  - 第二批: 基本面+估值
- 如果搜索量较小（已有人汇总页面），2-3条 broad search 即可覆盖全部6个模块

### 搜索工具选择
- ❌ 不要用 `curl` 直接请求 Google — Google 会返回 400 错误（反爬）
- ✅ 使用 `web_search` 工具（如果有）或 `web_extract`（直接抓指定页面）
- ✅ 优先抓已知数据聚合页（捷利交易宝、AAStocks 的 IPO 专页）
- ✅ 中文关键词效果优于英文

### 任务边界重叠 (Overlap)
一个搜索任务往往能覆盖多个模块的数据：
- **孖展搜索** → 通常同时返回 招股价/每手/入场费/招股时间/基石 等信息
- **基本面搜索** → 通常包含 募资用途/行业地位/客户集中度
- 不需要为每个模块独立建子任务 — 合并搜索方向可减少任务数

## 用户交流节奏

- 用户偏好极简交流，不耐烦长时间等待
- 数据搜集阶段如果超过30秒，发送一句进度提示（如「数据收集中，稍等」）
- 不要在中间阶段输出半成品或询问确认，一次性产出完整报告

## 网络兜底策略 (Critical)

⚠️ **当所有网络搜索都失败/超时时，禁止卡死循环搜索！** 执行以下兜底：

1. **搜1-2次就评估** — 如果首次搜索失败（curl timeout / search error），最多重试1次不同方式
2. **如果还是失败** — 立即输出完整的结构化模板（6个模块），所有数据项标注 `[暂未搜索到数据]`
3. **保存并发送** — 即使数据为空，也要生成 .md 文件并通过 MEDIA 发送给用户
4. **在备注中说明** — 第一行注明「⚠️ 当前网络无法连接数据源，以下为模板框架」

**搜索工具选择：**
- ❌ 不要用 `curl` 直接访问搜索引擎 — WSL 环境对 curl 搜索引擎请求反爬严格
- ✅ 使用 `web_search` 工具（如果有）
- ✅ 使用 `browser_navigate` 访问捷利交易宝（TradeGo）、AAStocks 等港股数据站
- ✅ 使用 `web_extract` 抓取指定URL的纯文本内容

**从飞书记话触发时：**
- 先在飞书记话中回复一条「正在搜索数据，稍等…」作为处理中提示
- 如果搜索超时，用户至少知道 bot 在工作而非无响应

## SOUL.md 自动触发机制

为了让港股IPO分析在 **所有平台**（飞书私聊、飞书群聊、CLI）自动触发，已向 `~/.hermes/SOUL.md` 添加规则：

- 当检测到港股IPO股票名称/代码时，**必须先加载此技能** (skill_view)
- 如果网络搜索失败，**不允许卡死** — 输出带「数据待补充」标记的结构化模板
- 通过飞书对话时，用 MEDIA 协议将生成的 .md 文件发回

SOUL.md 会加载到每个新会话的系统提示中，因此无需为每个平台单独配置。

- 数据可能随时更新，务必在每次回答时重新搜索最新数据
- 孖展数据截止时间对判断热度至关重要，标注截止时间
- 实操结论仅基于公开信息和历史数据，不构成投资建议，可在最后加上免责声明
- 如果用户输入的是市场整体情况（如"最近有什么新股"），调整为IPO日历/新股列表模式
